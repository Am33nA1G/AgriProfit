<objective>
Build the district harmonisation foundation: create the `district_name_map` database table and populate it using state-scoped RapidFuzz fuzzy matching across all four datasets (price, rainfall, weather, soil). This is the cross-dataset join key that every downstream ML feature depends on.

Purpose: Phase 2, 3, 4, 5, 6 all require joining price data to rainfall, weather, and soil datasets by district. Without a harmonised district name map, these joins either fail or produce severely degraded coverage (47.5% accuracy with global matching vs 93.1% with state-scoped matching per IDInsight empirical study).

Output: `district_name_map` table populated with all rainfall, weather, and soil district variants mapped to canonical price dataset district names; backend/app/ml/ module created; harmonisation script committed and tested.
</objective>

district_name_map table schema (from research):
```sql
id SERIAL PRIMARY KEY
source_dataset VARCHAR(20)   -- 'rainfall', 'weather', 'soil'
state_name VARCHAR(100)      -- canonical state name from price dataset
source_district VARCHAR(200) -- name as it appears in the source dataset
canonical_district VARCHAR(200) NULLABLE  -- matched price dataset name; NULL if unmatched
match_score FLOAT NULLABLE   -- RapidFuzz WRatio score 0-100
match_type VARCHAR(20)       -- 'exact', 'fuzzy_accepted', 'fuzzy_review', 'unmatched'
created_at TIMESTAMPTZ       -- server_default now()
UNIQUE (source_dataset, state_name, source_district)
INDEX: (state_name, canonical_district)
INDEX: (match_type)
```

Data column names (verified by direct inspection in research):
- Price parquet: columns = [date, commodity, commodity_id, state, state_id, district, district_id, price_min, price_max, price_modal, category_id, entity_id]
- Rainfall parquet: columns = [STATE, DISTRICT, year, month, rainfall] — note UPPERCASE
- Soil CSVs: columns = [cycle, state, district, block, nutrient, high, medium, low]
- Weather CSV path: "data/weather data/india_weather_daily_10years.csv" — inspect column names at runtime; fail fast if district column absent
