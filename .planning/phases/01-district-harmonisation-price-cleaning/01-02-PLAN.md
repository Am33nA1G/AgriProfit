<objective>
Build the price cleaning pipeline: create the `price_bounds` table, compute per-commodity IQR winsorisation bounds from the full 25M-row price parquet, flag and cap outlier rows (CV > 500% anomalies), and store bounds so the same caps apply consistently at inference time.

Purpose: The price parquet contains confirmed corrupt rows — Guar at CV 23,284%, Cumin Seed at 22,214%, Bajra at 9,413%. These unit-corruption outliers (per-kg prices in per-quintal fields) will corrupt every downstream feature, rolling statistic, and model if not capped before feature engineering begins. The `price_bounds` table is the persistent contract that all future ML pipeline steps use to apply consistent winsorisation.

Output: `price_bounds` table with one row per commodity (314 rows), each containing IQR-based lower_cap and upper_cap, outlier_count, and total_count. Clean prices pipeline script committed and tested.
</objective>

price_bounds table schema (from research data contract):
```sql
id SERIAL PRIMARY KEY
commodity VARCHAR(200)          -- commodity name as in price parquet
commodity_id INTEGER            -- from price parquet commodity_id column
q1 NUMERIC(12,2)               -- 25th percentile
q3 NUMERIC(12,2)               -- 75th percentile
iqr NUMERIC(12,2)              -- Q3 - Q1
lower_cap NUMERIC(12,2)        -- max(0, Q1 - 3*IQR)  — prices cannot be negative
upper_cap NUMERIC(12,2)        -- Q3 + 3*IQR
median_price NUMERIC(12,2)     -- median modal price
outlier_count INTEGER           -- rows outside [lower_cap, upper_cap]
total_count INTEGER             -- total rows for this commodity
computed_at TIMESTAMPTZ        -- server_default now()
UNIQUE (commodity)             -- one row per commodity
```

IQR winsorisation formula (from research Pattern 2):
```python
q1 = series.quantile(0.25)
q3 = series.quantile(0.75)
iqr = q3 - q1
lower_cap = max(0.0, q1 - 3 * iqr)  # clamp to non-negative
upper_cap = q3 + 3 * iqr
```

Immutability principle (from coding-style.md and research anti-patterns):
- NEVER overwrite `price_history.modal_price` in the database
- The script produces `is_outlier` flag and `modal_price_clean` columns on the in-memory DataFrame
- price_bounds table stores the caps; downstream feature code applies them at read time
